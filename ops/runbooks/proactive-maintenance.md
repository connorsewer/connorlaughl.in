# Proactive Maintenance

## Goals

- increase continuity
- reduce repeated explanation
- turn recurring work into structure
- stay helpful without becoming noisy

## Daily

- write important events and decisions to `memory/YYYY-MM-DD.md`
- capture durable preferences or standing facts in `MEMORY.md`
- note repeated friction that may deserve graph promotion

## Every few days

- review recent daily memory files
- promote reusable concepts into `graph/nodes/`
- create or improve MOCs when a topic cluster forms
- convert stable sequences into `ops/runbooks/`

## Weekly

- review `graph/index.md` and major MOCs
- split oversized nodes
- remove redundancy
- identify one candidate for automation, reminder, or workflow improvement

## Heartbeat guidance

Use for batched low-urgency checks and maintenance.

Examples:
- review important inbox/calendar changes
- memory cleanup
- graph promotion
- project status checks

## Cron guidance

Use for exact-time reminders and isolated recurring tasks.

Examples:
- time-specific reminders
- deadline nudges
- exact scheduled summaries

## Promotion ladder

- one-off event -> daily memory
- durable fact -> `MEMORY.md`
- reusable idea -> `graph/nodes/`
- topic overview -> `graph/mocs/`
- repeated sequence -> `ops/runbooks/`
- timed recurring task -> cron or heartbeat checklist
