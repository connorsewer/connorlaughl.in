# Skill Graph Design

A skill graph should be traversable before it is exhaustive.

## Core practices

- [[../nodes/atomic-notes-with-descriptions-outperform-monoliths]] — use small standalone notes so retrieval and recombination stay cheap
- [[../nodes/progressive-disclosure-is-a-capability-multiplier]] — decide what to read through titles, descriptions, and MOCs before loading full files
- [[../nodes/maps-of-content-steer-attention]] — guide traversal with opinionated topic maps rather than raw directories
- [[../nodes/links-should-carry-semantic-meaning]] — embed links in prose that explains why the destination matters

## Missing areas

- example-driven node template patterns
- graph evolution heuristics for when a topic should become its own MOC

## Anti-patterns

- giant single-file doctrine
- links as random directories rather than meaningful prose
- no descriptions in frontmatter
- dumping raw transcripts directly into primary context
