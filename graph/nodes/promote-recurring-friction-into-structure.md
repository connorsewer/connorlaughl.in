# Promote Recurring Friction Into Structure

If something recurs, it probably deserves a file.

Examples:

- a repeated explanation becomes a graph node
- a recurring decision area gets a MOC
- a common failure gets a troubleshooting note
- a reliable sequence becomes a checklist or skill
